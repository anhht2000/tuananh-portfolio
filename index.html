Web App for Headout
At Headout, ​I was given the opportunity to work on the Front end side of the progressive web apps. The main goal was to increase the ROI using enhanced UI/UX. The projects varied along the line. The project follows the same architecture as Facebook's create-react-app which is an starter project for React which also uses Webpack which is extensible for using Redux.

Task 1 - Referral Page

For this task, I was already assigned a week to learn about how the code works and the project structure. The app was fully written in React, Redux ​and Node. It took me around 3 weeks to complete this task wherein I had to implement the newly created designs for the desktop and the mobile web app. The backend APIs were written in Java.​ I had to use JSX ​for writing the templates, LESS ​as the pre-processor for stylesheets and thunks as the middleware ​for API calls. The effort was minimized using the Module Bundler, Webpack​. Its main purpose is to bundle JavaScript files for usage in a browser, and it is also capable of transforming, bundling, or packaging just about any resource or asset. The app was server side rendered which is why React is famous. The server side rendering was implemented using NodeJS

Task 2 - Categorisation

The project requirements were the same. All I had to do was to implement the new design using the same libraries as above. Previously, the categories were not nested. What nesting means, is giving the categories a new structure similar to a tree with interlinked nesting too. The backend was supposed to implement the logic and I was supposed to showcase it. Through this, the user can now see on which categorical level he was and would be able to navigate through the web app without any hassle. It would prove to increase the conversion rate. While working on this I was able to write a better code for the header which is basically the navbar. For implementing this, I had to fix a lot of things in the web app as the root structure for the categories were changed. I worked on the header, i.e. the navbar, the category filter and make a new super category page which is the root category. This was to be done for both desktop and mobile.

Task 3 - New Checkout Flow

For the sole purpose of enhancing the user experience the designer came up with a new way of procedure for booking any event. The design changes were massive and took a lot of effort to restructure the code. So to implement this we proceeded to implement only on mobile first with a AB test​ by showing it to 50% of the users. The checkout flow consisted of getting the user details like name, email and mainly the card details. This had to work along with the already existing code, so had to write modular code for this. Web App for XPrep
The app focused on creating a dashboard for tutors. It was done using a custom starter project which used AngularJS, GulpJs, etc. I worked on this app during my internship in summer '16 at XPrep.

The whole of design and the frontend along with the integration with backend was done by my. I am that kind of person who designs while coding. So a basic setup or wireframe is laid out, and I start coding it. Changing CSS here and there. Creating new elements and interactions using JavaScript.

I used Angular-Material for the implementation of Material Design. This app was completely mobile responsive.

I got to learn about Gulp from a very informative article from CSS-Tricks. Using this, I created my own custom tasks which were used as a standard in other starter projects. I used browser-sync for live-reloading. I didn't use jQuery because I read an article on why we shouldn't use jQuery with AngularJS. I used Bower as the package manager. The project was obviously git tracked.

I consumed the APIs in my services and used them modularly in my controllers. We used token based authentication for the app. I used UI-Router instead of ng-router due to its various advantages like the use of state. The main reason was the fact that I wanted to use sub-states or nested states. I also implimented OTP generation for phone for authentication.

The website is live at xprep.co.

Status App

The constant need of tracking the status of app services, systems and servers was the main idea behind this app. The app served a tool to check if any particular service or system is up and running or down. It also had a feature for emailing the admin if anything goes down.

This was one of the first projects I picked up in college. For this I created the wireframes and designs on Illustrator. This is the only app in which I used Illustrator for this purpose. For the creating apps nowadays, I create a basic layout in my head or paper and start coding and design while coding it.

The app worked solely on Sockets. What it does was listens to events or emits them. So if any event, like a system is down is emitted and this is listened in the frontend and the useful action is taken. The status was updated every 30 minutes ny pinging the service or the system.

It also had a feature for force updating the status, in which it forcefully updated the status for the selected services or systems. This app was made for the group I am in called Information Management Group, IIT Roorkee.

Chat module for Tagbin

This app served as an instant messaging module for Tagbin. It had single and multiple user chats. The web app was made already in Angular so this module was to be made in Angular too.

I didn't design anything here because we used a template project in which chat module was provided. I needed to make this working. We used StropheJS as the library for integrating the module with the backend.

StropheJS worked on XMPP Protocol, which is used by various instant messengers such as WhatsApp. XMPP works on messaging and presence objects using XML for exchanging data between users. This was used alongside AngularJS.

It already ahd factories and services written. So I needed to integrate them with the controllers. I needed to understand the code too for this. The app was completed only with the help of the mentor. Without him, I couldn't have managed to pull off this app.
